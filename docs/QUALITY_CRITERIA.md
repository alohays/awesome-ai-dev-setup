# Quality Criteria

Awesome lists are curations, not collections. Every entry must meet a quality bar to maintain the value of this list for the community.

## Inclusion Criteria

All new submissions must meet **every** criterion below:

### 1. Activity

- At least **1 commit in the past 12 months**.
- Demonstrates active or stable maintenance.

### 2. Documentation

- Must have a README containing:
  - What the project does
  - Installation instructions
  - Usage examples or getting started guide

### 3. Public Accessibility

- Must be publicly accessible on GitHub (or similar platform).
- No private repos, no broken links.

### 4. Relevance

- Must be directly relevant to AI-assisted development.
- Valid categories: agent templates, config/rules, skills/plugins, CLI scaffolding, cross-platform tools.
- General-purpose AI/ML libraries do not qualify.

### 5. Originality

- Must not be a fork without significant original work.
- "Significant" means substantial new features, different scope, or meaningful adaptation beyond the upstream project.

### 6. License

- Must include a license file in the repository.
- Open-source licenses preferred (MIT, Apache 2.0, CC0, etc.).

## Removal / Deprecation Policy

Entries will be removed when any of the following conditions are met:

- **Archived or deleted**: Repository has been archived or removed by its owner.
- **Inactive**: No commits for **18+ months** (longer grace period than the 12-month inclusion threshold).
- **Abandoned**: Project explicitly abandoned by maintainer (stated in README or issues).
- **License change**: License changed to non-open-source.
- **Scope drift**: Project scope pivots away from AI-assisted development.

### Removal Process

1. A maintainer opens an issue tagged `removal` with evidence.
2. 14-day grace period for community response.
3. Entry removed via PR after grace period if no valid objection.

## Review Cadence

- **Quarterly reviews** in January, April, July, and October.
- All entries are checked against current inclusion criteria.
- Entries failing criteria are flagged for the removal process.
- Review results are published as a GitHub issue for transparency.
